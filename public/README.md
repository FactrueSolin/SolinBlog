# public 目录说明

## 目录用途
`public/` 用于存放对外公开的静态资源文件（如图片、纯静态页面等）。这些文件会被直接映射到站点的 URL 路径中，供浏览器或其他客户端访问。

## URL 映射规则
该目录下的文件路径会与 URL 路径一一对应：

- 目录结构保持不变
- 文件名与扩展名保持不变

**规则公式：**

```
public/<path>  =>  /<path>
```

## 使用示例

| 文件路径 | 访问 URL |
| --- | --- |
| `public/Solin.png` | `/Solin.png` |
| `public/images/logo.svg` | `/images/logo.svg` |
| `public/docs/intro.html` | `/docs/intro.html` |

## 特殊文件说明（可选）

以下文件属于**可选的特殊配置文件**：如果存在，会自动被页面使用；如果不存在，页面会使用默认样式。

- `public/icon.png`：网站 favicon 图标
- `public/light.png`：明亮模式背景图片
- `public/night.png`：暗黑模式背景图片

## build.rs 自动图片转换说明

在编译阶段，`build.rs` 会自动处理 `public/` 目录下的特殊图片文件，用于生成标准的 PNG 版本。

- 支持的特殊文件名：`icon`、`light`、`night`
- 若存在非 PNG 格式（如 jpg、webp、gif 等），会自动转换生成对应的 `.png` 文件
- 原始文件会被保留，不会被删除
- 如果对应的 `.png` 已存在，则跳过不转换
- 转换失败只会输出警告，不会中断编译

## 注意事项

1. **仅放置可公开的内容**：该目录中的文件会被直接访问，请勿放置敏感信息（如密钥、配置文件、私有数据）。
2. **避免覆盖路由**：若应用层有同名动态路由或 API 路径，需确认是否会被静态文件优先匹配。
3. **文件名区分大小写**：在类 Unix 环境中路径区分大小写，URL 访问需严格匹配文件名。
4. **禁止执行脚本**：放置脚本文件时请确认服务器不会以可执行方式处理，避免安全风险。
